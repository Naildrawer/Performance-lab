Heap
Это область памяти Java, где хранятся объекты.
Все созданные объекты помещаются в Heap.
Размер памяти можно ограничить настройками JVM.
Heap делится на поколения для эффективной очистки.
Когда памяти не хватает, запускается сборщик мусора.
Переполнение Heap приводит к OutOfMemoryError.

Garbage Collector это сборщик мусора Java.
Он удаляет объекты, которые больше не используются.
Освобождает память автоматически.
Работает периодически во время работы приложения.
Может вызывать небольшие паузы.
Позволяет разработчику не освобождать память вручную.

Thread Dump Это снимок всех потоков приложения.
Показывает их текущее состояние.
Помогает искать зависания и блокировки.
Содержит стек вызовов каждого потока.
Используется при анализе производительности.
Часто снимается командой jstack.1

Heap Dump
Это снимок содержимого Heap памяти.
Показывает все объекты, находящиеся в памяти.
Используется для поиска утечек памяти.
Можно увидеть самые большие объекты.
Анализируется в специальных программах.
Часто сохраняется в файл hprof.

HikariCP
Это библиотека для пула соединений с базой данных.
Хранит готовые соединения для повторного использования.
Ускоряет работу приложения.
Уменьшает затраты на создание новых соединений.
Считается одним из самых быстрых пулов соединений для Java.
Часто используется в Spring Boot по умолчанию.
